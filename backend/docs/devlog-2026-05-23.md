## 四、关键技术决策

1. **类型编码映射**: API 使用英文编码 (stock/mixed/index/etf/…)，通过 `FUND_TYPE_PREFIX_MAP` 转换为 DB 中文前缀 (`LIKE "股票型%"`)，兼容 DB 中 "股票型"、"股票型-偏股" 等子类型
2. **AkShare 版本兼容**: AkShare 函数名因版本差异有变化 (`fund_estimate_em` → `fund_value_estimation_em`, `stock_board_industry_em` → `stock_board_industry_spot_em`)，通过 mock 测试覆盖而非硬依赖特定版本
3. **动态列名解析**: `fund_value_estimation_em` 返回的列名包含日期 (如 `2026-05-22-估算数据-估算值`)，用 `_parse_estimate_row()` 按后缀模式匹配而非硬编码列名
4. **重采集机制**: 所有 batch_upsert 返回 `(added, updated)` 二元组，CollectResult 正确区分新增/更新计数
5. **调度器**: 使用 cron 表达式替代 interval 触发器，避免 interval 在不同采集频率下资源浪费；非交易日自动跳过实时采集任务
